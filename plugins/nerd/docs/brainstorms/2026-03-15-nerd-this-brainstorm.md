# /nerd-this — Context-Scoped Experiment Discovery

## What We're Building

A new standalone command `/nerd-this` that generates nerd experiments scoped to what you're currently working on — not the whole codebase. It infers scope from session context (files touched, branch diff, conversation history) or accepts an explicit topic string. It presents discovered research themes as selectable options, then launches broad research across selected themes.

## Usage

```
/nerd-this                                    # infer scope from session context
/nerd-this pi agent as a replacement to ACP   # explicit topic narrows discovery
```

## How Scope Discovery Works

Three signals, triangulated and ranked by recency/frequency:

1. **Files read/edited in this session** — tool use history shows what you've been touching
2. **Git branch diff** — files changed on this branch vs main reveal the spike's footprint
3. **Conversation content** — what you've been discussing, planning, deciding

With an explicit topic string, these signals are filtered/weighted toward relevance to that topic. Without one, the signals alone define scope.

## Output: Theme-Based Selection

Unlike `/nerd` which presents individual parameters, `/nerd-this` groups discoveries into **themes** — coherent research areas derived from the scoped context.

Example output for `/nerd-this pi agent as a replacement to ACP`:

```
Found 4 research themes in your current work:

  [x] Agent lifecycle management
      IPC socket timeouts, process spawn config, graceful shutdown sequences
      12 tunable parameters across 6 files

  [x] Message serialization pipeline
      Codec buffer sizes, streaming chunk thresholds, backpressure limits
      8 tunable parameters across 3 files

  [ ] MCP tool registration
      Tool discovery polling interval, capability cache TTL
      4 tunable parameters across 2 files

  [x] Migration compatibility layer
      ACP fallback thresholds, feature flag rollout percentages
      6 tunable parameters across 4 files

Select themes to research (all selected by default):
```

Selected themes feed into the existing nerd pipeline (plan-reviewer → lab-tech → experiment-executor → report-compiler) with all parameters in those themes included by default. The value is breadth — lots of research, not selective parameter picking.

## Why This Approach

- **Thematic grouping** matches how spikes are structured — you think in terms of "agent lifecycle" not "line 92 threshold"
- **Session inference** means zero friction — just `/nerd-this` and go
- **Explicit topic** option disambiguates when session context is noisy
- **Standalone command** keeps `/nerd` (full codebase scan) simple and unchanged
- **Broad by default** — all themes selected, user deselects what's out of scope

## Key Decisions

- **Standalone `/nerd-this` command**, not a flag on `/nerd` — different intent deserves its own entry point
- **Thematic grouping over parameter-level selection** — the value is running lots of research, not cherry-picking
- **Three-signal triangulation** (session files, git diff, conversation) — no single signal is reliable alone
- **All themes selected by default** — encourages broad research, user opts out rather than opts in
